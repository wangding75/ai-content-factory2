# Iteration 14 — WorkflowRun 运行时 — Closed Loop

## 1. 信息架构闭环

```text
项目概览 / 具体业务环节
        ↓ 携带 projectId 与 stage
流程中心 · 运行记录
        ↓
运行详情
```

原则：

- 流程中心是唯一运行记录中心；
- 流程中心不包含“工作流配置”Tab；
- 全局工作流配置继续位于全局设置；
- 项目设置只负责四环节绑定；
- 项目侧只显示摘要、最近运行和快捷入口。

## 2. 项目侧运行记录闭环

项目概览 P14_10 展示：

- 总运行次数；
- 运行中；
- 最近失败；
- 最近运行；
- 最近三条运行。

用户动作：

| 动作 | 结果 |
|---|---|
| 查看全部运行记录 | 进入 `/workflow-runs?projectId={projectId}` |
| 查看该环节记录 | 进入 `/workflow-runs?projectId={projectId}&stage={stage}` |
| 查看详情 | 进入 `/workflow-runs/{runId}` |
| 清除项目筛选 | 回到全局 `/workflow-runs` |

P14_02 必须明确展示“项目：星海纪元”筛选条件，用户可以清除。

## 3. 发起运行闭环

业务页面点击“运行工作流”。

### 3.1 已绑定且可运行

1. 读取项目当前环节绑定；
2. 检查工作流和连接状态；
3. 展示 P14_08 运行确认弹窗；
4. 用户确认后提交 `POST /api/v1/workflow-runs`；
5. API 创建 `queued` 运行和非敏感快照；
6. 前端直接跳转 `/workflow-runs/{runId}`；
7. Worker 推进运行状态。

### 3.2 未绑定

1. 不创建运行；
2. 展示 P14_09 未绑定提示弹窗；
3. 文案动态显示当前项目和环节；
4. 点击“前往工作流绑定”进入项目设置；
5. 完成绑定后返回业务页面继续运行。

P14_09 不是独立页面，也不替换业务页面内容。

### 3.3 已绑定但不可运行

- 连接未验证、未启用或异常；
- 工作流未验证、未启用或配置失效。

系统不创建运行，展示安全错误与配置入口。不得把“已绑定”误表示为“可运行”。

## 4. Runtime 闭环

```text
queued → running → succeeded
queued → cancelled
running → failed
running → cancelled
```

执行步骤：

1. 创建运行任务；
2. 进入执行队列；
3. 调用工作流；
4. 校验输出；
5. 保存领域结果；
6. 写入终态和事件。

页面刷新后从 WorkflowRun 和 Event 恢复，不依赖浏览器内存状态。

## 5. 运行详情闭环

运行详情统一使用全局上下文：

- 面包屑：流程中心 / 运行记录 / Run Number；
- 所属项目为可点击属性；
- 返回按钮为“返回运行记录”；
- 返回时保留来源筛选参数。

状态：

- P14_05 成功：展示结果摘要和业务结果入口；
- P14_06 运行中：展示当前节点、刷新和取消；
- P14_07 失败：展示业务可理解错误、处理建议、重新运行和配置入口。

技术详情默认折叠；不得直接展示凭证、请求 Header、完整上游响应或堆栈。

## 6. 取消与重试闭环

### 取消

- 仅 `queued/running` 可取消；
- 使用 `expectedVersion`；
- 取消成功进入 `cancelled`；
- Adapter 无法中断远端执行时，仍需记录取消请求与实际结果，不伪造成功中断。

### 完整重试

- 原运行保留；
- 创建新 Run Number；
- 建立 `retryOfRunId`；
- 可以使用原快照或当前有效配置；
- 新运行从 `queued` 开始。

### 领域提交重试

- 只在外部执行已成功、领域提交失败时开放；
- 不重新调用 n8n；
- 使用已校验输出重新执行领域事务；
- 记录独立事件。

## 7. 列表状态闭环

- P14_01：全局有数据；
- P14_02：项目筛选；
- P14_03：全局空状态；
- P14_04：加载失败；
- Loading 使用局部 Skeleton，保持页面结构稳定；
- 列表失败不能伪装成空数据；
- 统计卡片和表格必须使用同一筛选口径。

## 8. n8n 与安全边界

- Worker 主动调用 n8n Webhook；
- n8n 通过 HTTP 响应返回结果；
- 不实现 callback server；
- 凭证只在 Worker 内短暂解密；
- 日志和数据库不保存凭证、Authorization Header 或原始上游响应；
- 错误必须归一化和脱敏；
- 输出 Schema 非法时不写领域结果。

## 9. 明确不做

- 流程中心工作流配置 Tab；
- 项目级重复运行记录页面；
- LLM Provider 直接调用；
- 分发平台绑定、验证或发布；
- n8n 可视化编辑；
- 自动重试编排；
- 其他 Workflow Adapter。
