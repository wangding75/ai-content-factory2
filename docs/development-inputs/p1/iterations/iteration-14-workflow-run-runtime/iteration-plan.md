# Iteration 14 — WorkflowRun 运行时与 n8n 执行闭环

## 1. 目标

在 Iteration 12 全局连接/工作流配置与 Iteration 13 项目四环节绑定基础上，实现：

1. n8n Connection Adapter 及连接、工作流的真实验证与启停；
2. `WorkflowRun` 创建、排队、执行、取消、成功、失败与重试；
3. Worker 异步调用 n8n Webhook，并通过 HTTP 响应接收结果；
4. 全局“流程中心”运行记录列表与运行详情；
5. 项目概览中的运行摘要、最近运行和快捷入口；
6. 从章节规划、内容生成、审核、改写页面触发运行；
7. 绑定快照、输入输出、事件时间线、错误脱敏与领域提交边界。

Iteration 14 的产品定位是：**项目侧发起与查看摘要，流程中心统一查看全部运行记录。**

## 2. 已冻结的信息架构

### 2.1 流程中心

- 路由：`/workflow-runs`
- 只承担全局运行记录管理；
- 不设置“工作流配置”Tab；
- 不复制全局设置中的连接和工作流配置能力；
- 支持按项目、业务环节、工作流、状态、时间和运行编号筛选。

### 2.2 运行详情

- 路由：`/workflow-runs/{runId}`
- 面包屑固定为：`流程中心 / 运行记录 / {runNumber}`；
- 项目是运行记录属性，可点击返回项目，但不作为详情页父级路由；
- 成功、运行中、失败使用同一详情结构。

### 2.3 项目侧

项目概览只展示：

- 总运行次数；
- 运行中数量；
- 最近失败数量；
- 最近运行时间；
- 最近三条运行；
- “查看全部运行记录”与环节级快捷入口。

跳转规则：

- 全部项目记录：`/workflow-runs?projectId={projectId}`；
- 指定环节：`/workflow-runs?projectId={projectId}&stage={stage}`；
- 项目侧不建设第二套运行记录列表。

### 2.4 业务环节触发

适用于：

- `chapter_planning`：章节规划；
- `content_generation`：内容生成；
- `review`：审核；
- `rewrite`：改写。

点击“运行工作流”后：

- 已绑定且可运行：展示运行确认弹窗；
- 未绑定：展示未绑定提示弹窗，并跳转项目设置的工作流绑定页；
- 确认成功创建运行后，直接跳转 `/workflow-runs/{runId}`。

## 3. 前置条件

- Iteration 12 已保存工作流连接和工作流配置；
- Iteration 13 已保存项目四环节绑定；
- 绑定关系不等于可执行，运行前必须验证工作流与连接状态；
- 运行使用创建时快照，后续配置变更不影响历史记录。

## 4. 运行闭环

业务页面点击运行 → 检查项目绑定 → 检查工作流与连接可运行 → 展示确认弹窗 → 创建 `queued` 运行与快照 → Worker 领取 → `running` → 调用 n8n Webhook → 校验输出 → 领域提交 → `succeeded`。

失败路径：

- 未绑定：不创建运行，展示 P14_09；
- 连接/工作流未就绪：不创建运行，返回安全错误和配置入口；
- HTTP、超时、Adapter 或输出 Schema 错误：运行进入 `failed`；
- 用户取消：合法状态进入 `cancelled`；
- 领域提交失败：保留外部输出，允许只重试领域提交；
- 完整重试：创建新运行，并保留原运行。

不建设 callback server；n8n 通过同步 Webhook HTTP 响应返回执行结果。
