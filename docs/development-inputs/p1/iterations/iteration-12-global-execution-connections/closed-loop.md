# Iteration 12 — 全局配置中心 V2 — Closed Loop

## 1. 入口

用户从共享侧边栏进入“设置”，一级 Tab：

- LLM 配置；
- 工作流配置；
- 分发平台配置。

工作流配置包含：

- 连接；
- 工作流。

## 2. 通用配置闭环

空状态 → 点击工作区右上角添加按钮 → 填写表单 → 本地校验 → 服务端字段校验 → 保存 → 关闭抽屉 → 刷新列表 → 查看详情或编辑。

本迭代的“完成”只表示配置已安全保存，不表示第三方已验证或可运行。

## 3. LLM 配置闭环

- 添加 Provider 类型、名称、Base URL、默认模型、超时和 API Key；
- API Key 加密保存；
- 编辑时不回显 API Key；
- 未填写新 API Key 时保留原值；
- 保存后显示“未接入”；
- 验证、模型发现和启用控件禁用，并说明将在后续迭代开放。

## 4. 连接配置闭环

- 新增态允许选择连接类型；
- 当前连接类型为 n8n；
- 编辑态连接类型只读；
- 保存 Base URL、认证方式、凭证、超时和类型专属字段；
- 不访问 n8n；
- 保存后显示“未接入”；
- 验证和启用控件禁用。

## 5. 工作流配置闭环

- 选择关联连接；
- 系统自动推导连接类型和工作流类型；
- 用户不能独立修改工作流类型；
- 保存适用环节、类型专属引用、契约版本、默认参数和备注；
- 修改关联连接后重新推导类型；
- 连接类型变化时清空旧类型专属字段；
- 不验证 Workflow ID 或 Webhook Path；
- 保存后显示“未接入”。

工作流列表不得出现没有关联连接的系统内置记录。

## 6. 分发平台配置闭环

- 选择平台类型；
- 填写账号标识、认证方式、凭证和平台专属字段；
- 凭证加密保存且编辑不回显；
- 不验证平台、不执行 OAuth、不发布内容；
- 保存后显示“未接入”；
- 验证和启用控件禁用。

## 7. 异常闭环

- 本地字段错误：在字段附近显示中文错误；
- 服务端 Schema 错误：保留表单并定位字段；
- 名称冲突：提示更换名称；
- 乐观锁冲突：提示配置已被修改，刷新后重新编辑；
- 保存失败：不关闭抽屉，不丢失输入；
- 凭证加密失败：整体事务回滚；
- 页面刷新：从真实 CRUD API 恢复列表和详情；
- 不存在第三方网络错误状态，因为本迭代禁止出站调用；
- 审计记录不得包含敏感值。

## 8. 后续闭环

- Iteration 13 保存项目四环节绑定关系；
- Iteration 14 对 n8n 连接和工作流执行真实验证、启用和运行；
- Iteration 15 或首个直接 LLM 业务迭代接入 LLM Provider；
- 分发平台真实适配在未来发布迭代完成。
