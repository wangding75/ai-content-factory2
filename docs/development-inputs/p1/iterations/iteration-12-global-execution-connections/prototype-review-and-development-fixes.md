# Iteration 12 原型评审结论与开发修复项

## 1. 评审结论

本批 Stitch 原型继续作为开发参考，不再重新生成。

> **有条件通过，所有问题在开发阶段强制修复。**

## 2. 原型问题

### P0-1：工作流列表仍是旧字段结构

开发修复：

- 移除独立“类型”列；
- 增加“关联连接”和“连接类型”；
- 每个工作流必须关联连接；
- 工作流类型由关联连接自动推导；
- 删除“系统内置”且无连接的示例。

### P0-2：新增连接时连接类型被禁用

开发修复：

- 新增态：连接类型必填且可选择，当前只有 n8n；
- 编辑态：连接类型只读且后端拒绝修改；
- 新增和编辑复用抽屉，但必须按模式切换控件状态。

### P0-3：抽屉背景错误

开发修复：

- 连接抽屉使用真实连接列表工作区；
- 工作流抽屉使用真实工作流列表工作区；
- LLM 和分发平台抽屉同样使用各自真实父列表页；
- 不实现原型中的虚构显示器场景或错误背景。

### P0-4：页面壳层风格不统一

开发修复：

- 复用仓库现有侧边栏、顶部栏、页面头部、面包屑和用户信息组件；
- 只开发全局设置标题下方工作区；
- 不根据每个原型分别实现页面壳层。

### 重复添加按钮

开发修复：

- 空状态和列表状态只保留工作区标题栏右上角的添加按钮；
- 空状态卡片内不实现第二个按钮；
- 状态切换时按钮位置不变化。

## 3. Iteration 12 后端范围

Iteration 12 后端只覆盖：

- LLM Provider 配置 CRUD；
- 通用连接配置 CRUD；
- 工作流配置 CRUD；
- 分发平台配置 CRUD；
- 类型目录和动态字段 Schema；
- 凭证加密存储；
- 工作流类型推导；
- 乐观锁、幂等和安全审计。

Iteration 12 明确不覆盖：

- 真实连接验证；
- 启用或停用；
- LLM 模型发现；
- LLM 直接调用；
- n8n Adapter；
- n8n Webhook 调用；
- 工作流运行；
- 平台验证、OAuth 和发布；
- 任何第三方出站网络请求。

UI 中的验证和启用相关控件保留位置，但显示为“未接入”并禁用。

- 不得实现多套页面壳层；

- 不得允许客户端独立修改工作流类型；
- 不得回显凭证；
- 不得添加删除入口；
- 不得在 Iteration 12 实现 Adapter；
- 不得在 Iteration 12 发起第三方请求；
- 不得实现 callback server。
