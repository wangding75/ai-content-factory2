# Iteration 13 — 项目四环节工作流绑定 — 验收标准

## 1. 绑定闭环

- [ ] 四个环节拥有独立绑定；
- [ ] 每个项目和环节最多一个当前绑定；
- [ ] 可以选择 Iteration 12 已保存的工作流；
- [ ] 保存默认参数；
- [ ] 列表和抽屉展示工作流、连接类型和接入状态；
- [ ] `not_connected` 工作流可以绑定；
- [ ] 未接入绑定显示“尚未接入执行能力”；
- [ ] 页面刷新后绑定状态恢复；
- [ ] OpenAPI、实现和 UI 字段一致。

## 2. 范围边界

- [ ] API 只包含绑定 GET 和 PUT；
- [ ] 不包含 validate、enable、disable、run 或 retry；
- [ ] 不实现第三方 Adapter；
- [ ] 不访问 LLM、n8n 或分发平台；
- [ ] 不创建 WorkflowRun；
- [ ] 不创建绑定快照；
- [ ] 不显示运行日志、失败诊断或重试入口；
- [ ] 自动测试证明保存绑定不产生第三方请求。

## 3. 数据和异常

- [ ] 工作流不存在或归档时阻止保存；
- [ ] 参数按环节 Schema 校验；
- [ ] 乐观锁冲突处理正确；
- [ ] 失败不产生部分绑定数据；
- [ ] 审计不包含凭证或运行数据。

## 4. UI

- [ ] 默认中文环境下用户可见文案全部使用中文资源；
- [ ] 左侧一级菜单统一；
- [ ] 技术标识可以保留英文；
- [ ] 未接入状态和不可执行原因清晰；
- [ ] 原型中的运行状态文案不在本迭代实现。

## 5. 工程验收

- [ ] 先局部测试，再分组测试，最后一次总门禁；
- [ ] 未定位根因前禁止反复完整重跑；
- [ ] 类型检查、构建、API 测试和 E2E 通过；
- [ ] `git diff --name-status`、未跟踪文件和 `git status --short` 已记录；
- [ ] 独立 Code Review 完成。
