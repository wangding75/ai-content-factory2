# Iteration 13 — 项目四环节工作流绑定 — Closed Loop

## 1. 主入口

`项目工作区 → 设置 → 工作流绑定`

页面固定展示：

- 章节规划；
- 内容生成；
- 审核；
- 改写。

项目策划页和项目概览只提供动态快捷入口，不复制工作流绑定管理界面：

- 策划完成且仍有未配置环节：显示“配置项目工作流”；
- 概览页“下一步建议”在策划完成但绑定未完成时动态建议配置工作流；
- 点击后统一进入项目设置的工作流绑定页。

## 2. 用户动作与系统结果

| 用户动作 | 系统结果 |
|---|---|
| 打开工作流绑定 | 从真实 API 返回四个固定环节及当前绑定 |
| 选择工作流 | 只展示适用于当前环节的候选项 |
| 确认绑定 | 保存项目、环节与全局工作流的唯一关系 |
| 更换工作流 | 原绑定被新的工作流替换，其他环节不受影响 |
| 解除绑定 | 只删除当前项目当前环节的绑定，不删除全局工作流 |
| 刷新页面 | 从 PostgreSQL 恢复最新绑定状态 |

## 3. 状态语义

- **已绑定**：项目与全局工作流之间的关系已保存；
- **已启用/已停用**：全局工作流当前启用状态；
- **已集成/未接入**：全局工作流的集成状态；
- **连接异常**：关联连接当前不可用。

“已绑定”不等于已验证、已集成或可执行。

## 4. 异常闭环

- 候选加载失败：保留当前环节上下文并提供重试；
- 无可用工作流：说明需要在全局设置中创建并配置适用环节，提供跳转入口；
- 全局工作流后续停用、未接入或连接异常：保留现有绑定，显示原因，不自动解绑；
- 保存字段错误：在对应字段或选择区域显示；
- 409 冲突：保留用户当前选择，提供“加载最新配置”，不得静默覆盖；
- 保存失败：不得产生部分绑定或重复 Audit；
- 页面刷新：始终以服务端最新状态为准。

## 5. 明确不做

- 不验证、启用或停用全局工作流；
- 不执行 n8n、Webhook 或任何第三方调用；
- 不创建 `WorkflowRun`；
- 不保存运行快照、执行结果或重试记录；
- 不在业务执行页临时切换工作流。
