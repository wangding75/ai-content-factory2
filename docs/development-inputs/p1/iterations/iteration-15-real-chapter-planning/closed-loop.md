# Iteration 15 — 真实章节规划与候选批次闭环

> 业务状态机以 `business-rules.md`（`frozen_cf_15_01a`）为准。

## 1. 主入口

项目工作区 → 章节 Tab → 章节规划工作区。

页面同时展示：

- 当前章节列表；
- 当前活跃章节规划 Run；
- 候选批次数量；
- 待确认、已确认和已生成草稿统计；
- 候选批次入口。

## 2. 生成闭环

1. 用户点击“生成章节规划”。
2. 选择生成模式：
   - 完整章节规划；
   - 补充后续章节；
   - 指定范围生成。
3. 设置目标章节、重点故事线、上下文范围和附加要求。
4. 点击确定后进入预检；预检不创建任务、不调用外部执行器。
5. 预检报告分为：
   - 通过；
   - 警告但可继续；
   - 阻断。
6. 阻断时提供返回修改或前往配置入口。
7. 用户在通过报告中确认后，创建 `WorkflowRun(stage=chapter_planning)`。
8. 创建成功页显示 Run 编号、时间、模式、目标数量和 queued 状态。
9. 用户可返回工作区或进入全局运行详情。
10. 页面刷新后从持久化 Run 恢复 queued/running/failed 状态。

## 3. 结果入库闭环

1. 外部执行层返回规范化结果。
2. 服务端校验输出 Schema、章节编号、数量、引用和业务约束。
3. 校验全部通过后，在一个事务内创建：
   - 一个候选批次；
   - 全部候选章节；
   - 来源 Run、输入摘要和基线版本。
4. 任意候选非法或事务失败时：
   - 不保存任何候选；
   - 不保存半个批次；
   - Run 保留安全错误；
   - 用户通过运行详情重试。
5. Run succeeded 与候选结果消费成功相互独立；只有消费成功后，候选批次才进入 `ready`。

## 4. 候选处理闭环

候选批次列表
→ 进入批次详情
→ 筛选候选
→ 查看当前章节与新候选差异
→ 编辑候选
→ 单个采用、批量采用或放弃候选。

### 4.1 单个采用

- 新增候选：创建新的当前章节。
- 替换候选：以 `expectedVersion` 校验当前章节后创建新修订并更新当前章节。
- 无变化：允许跳过，不制造无意义修订。
- 过期冲突：禁止直接采用。

采用后：

- 当前章节状态为 `pending_confirmation`；
- 记录来源批次、候选和 WorkflowRun；
- 候选状态为 `adopted`；
- 不自动确认；
- 不自动创建正文。

### 4.2 批量采用

- 用户可选择多个候选。
- 系统必须先分类新增、替换、无变化和冲突。
- 每项在明确确认后以独立事务采用，并逐项返回 adopted、no_change、stale/conflict 或 failed 结果。
- 每个候选结果可追溯，批次统计同步更新。
- 批量采用不得静默覆盖已变化章节。

### 4.3 过期冲突

当 `candidate.baseChapterVersion != currentChapter.version`：

- 显示候选基线与当前版本；
- 提供重新对比；
- 允许编辑候选以适配最新上下文；只有以最新基线重算一致后才可回到 `pending`；
- 允许放弃候选；
- 禁止直接采用旧候选。

## 5. 批次放弃闭环

1. 用户在批次详情点击放弃批次。
2. 弹窗显示未采用、已采用和冲突数量。
3. 用户确认“已采用章节不会撤销”。
4. 提交放弃：
   - 未采用候选转为 `discarded`；
   - 批次转为 `abandoned`；
   - 已采用章节保持不变；
   - Run、批次和来源追溯保留。
5. 重复提交必须幂等。

## 6. 当前章节确认闭环

候选采用
→ 当前章节进入 `pending_confirmation`
→ 用户编辑、删除或批量选择
→ 复用 P0 批量确认
→ 选中章节转为 `confirmed`
→ “进入正文生产”入口开放。

未确认章节调用正文创建接口仍返回业务冲突。

## 7. 故事线闭环

- 故事线是递归树。
- 生成输入可选择父级并包含子分支。
- 章节规划结果负责实际章节落点。
- 故事线页面只读展示关联章节统计和候选推荐。
- 删除或失效父节点引用在预检阶段阻断。

## 8. 异常与恢复

| 场景 | 系统行为 | 用户恢复 |
|---|---|---|
| 未绑定工作流 | 不进入预检或创建 Run | 前往项目设置 |
| 执行集成不可用 | 预检阻断；本迭代不实现 Transport、Worker 或 Callback | `CF-14-N8N-Integration` 完成后重新预检 |
| 存在活跃冲突 Run | 预检阻断 | 查看运行详情、取消或等待 |
| 故事线引用失效 | 预检阻断 | 修复故事线 |
| 运行失败 | 不创建候选批次 | 查看详情并 Retry |
| 输出 Schema 非法 | 零候选写入 | Retry 新 Run |
| 候选采用版本冲突 | 不覆盖当前章节 | 重新对比、编辑或放弃 |
| 批次放弃重放 | 返回首次结果 | 无重复状态变化 |
| 页面刷新 | 重新读取持久化状态 | 无需重新发起 |
