# Iteration 15 — 验收标准

> 业务语义验收以 `business-rules.md`（`frozen_cf_15_01a`）为准。
> 数据结构、Migration 终态和事务边界以 `data-model.md`、`transaction-and-migration-design.md`（`frozen_cf_15_01b`）为准。

## 1. 契约与边界

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC01 | P0 继承 | 当前章节列表、编辑、删除、确认和正文门槛保持兼容。 |
| I15-AC02 | Runtime 复用 | 只复用 Iteration 14 WorkflowRun，不新增第二套 Run、Event、Cancel 或 Retry 模型。 |
| I15-AC03 | 执行边界 | 通用 n8n Transport/Worker/Callback 不在 Iteration 15 业务模块重复实现。 |
| I15-AC04 | 前后迭代边界 | Iteration 15 只交付章节规划；正文生成、审核、重写不进入本轮。 |

## 2. 预检

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC05 | 无副作用预检 | 预检不创建 Run、不调用外部执行器、不创建候选、不修改当前章节。 |
| I15-AC06 | 输入摘要 | 返回生成模式、章节范围、故事线、素材、伏笔、前序摘要和执行配置摘要。 |
| I15-AC07 | 警告 | 非阻断问题可显示并允许用户继续。 |
| I15-AC08 | 阻断 | 缺少绑定、执行集成不可用、活跃 Run 冲突、失效故事线引用或输入非法时不得创建 Run。 |
| I15-AC09 | 重新预检 | 返回修改后可重新预检，旧结果不能静默复用到不同输入。 |
| I15-AC09A | Token | Token 仅对同项目、同用户、同输入有效，10 分钟后失效；创建时复核关键状态。 |

## 3. Run 创建与恢复

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC10 | 创建任务 | 预检通过并确认后创建 queued WorkflowRun；stage 和 manual 来源由服务端固定。 |
| I15-AC11 | 幂等 | 同 Key 同载荷返回首次结果；同 Key 不同载荷冲突；不重复创建 Run。 |
| I15-AC12 | 页面恢复 | 刷新后可恢复 queued/running/failed 及 Run 详情入口。 |
| I15-AC13 | 活跃锁 | 同项目章节规划冲突 Run 的约束与错误码符合冻结契约。 |

## 4. 结果校验与原子写入

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC14 | 合法结果 | 一个成功 Run 只创建一个候选批次，候选数量和目标一致。 |
| I15-AC15 | 非法 Schema | 任意候选字段或引用非法时不创建批次或候选。 |
| I15-AC16 | 事务失败 | 数据库失败时不保存部分候选，允许专用恢复或新 Run 重试。 |
| I15-AC17 | 来源追溯 | 批次和候选可追溯至 WorkflowRun、绑定快照、生成输入和基线版本。 |
| I15-AC18 | 安全 | 不保存或返回密钥、原始上游响应、内部 URL、堆栈或原始幂等键。 |
| I15-AC18A | 消费幂等 | 同一 Run 重复消费不会创建第二个 Batch；Run succeeded 不等于消费成功。 |
| I15-AC18B | 数据终态 | Batch/Candidate/Revision 的来源、完整快照、版本、索引、保留和 Migration 设计符合 CF-15-01B；本任务不创建 Migration。 |

## 5. 候选批次

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC19 | 批次列表 | 支持项目、状态、来源和时间筛选；数量统计来自服务端。 |
| I15-AC20 | 批次详情 | 展示模式、范围、Run、候选数、已采用、待处理、冲突和放弃数量。 |
| I15-AC21 | 候选详情 | 展示当前章节、新候选、故事线、素材、伏笔、作用和上下文依据。 |
| I15-AC22 | 编辑候选 | 只有可处理候选可编辑；保存使用乐观锁。 |
| I15-AC23 | 差异对比 | 新增、删除、修改和保留内容可区分；技术数据不泄露。 |

## 6. 采用、冲突和放弃

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC24 | 新增采用 | 创建当前章节并记录修订与来源，状态为 pending_confirmation。 |
| I15-AC25 | 替换采用 | 正确 expectedVersion 下创建新修订并更新当前章节，旧内容可追溯。 |
| I15-AC26 | 采用不确认 | 采用后不自动 confirmed、不创建正文。 |
| I15-AC27 | 批量采用 | 只采用明确选择且无冲突的候选；冲突项保留待处理。 |
| I15-AC28 | 版本冲突 | 候选基线过期时禁止直接采用，提供重新对比、编辑和放弃。 |
| I15-AC29 | 放弃候选 | 放弃后不可采用，保留审计和来源。 |
| I15-AC30 | 放弃批次 | 未采用候选被放弃；已采用章节不回滚；操作幂等。 |
| I15-AC30A | 批量采用事务 | 每个 Candidate 独立事务并逐项返回；不得整批回滚、静默跳过或强制覆盖 stale。 |
| I15-AC30B | 数据库并发保证 | 活跃 Run、source Run Batch、Batch chapterNo、Revision 顺序和当前章节号均有数据库最终约束。 |

## 7. 当前章节和故事线回归

| ID | 场景 | 通过标准 |
|---|---|---|
| I15-AC31 | 当前章节编辑 | pending_confirmation 章节编辑刷新后一致。 |
| I15-AC32 | 当前章节删除 | 只允许删除冻结规则允许的章节状态。 |
| I15-AC33 | 批量确认 | 仅选中 pending_confirmation 转为 confirmed，重复提交不破坏状态。 |
| I15-AC34 | 正文门槛 | 未确认章节仍禁止进入正文；confirmed 章节允许进入 Iteration 16 入口。 |
| I15-AC35 | 故事线职责 | 故事线页面只读展示章节关联统计，不提供可编辑章节范围。 |
| I15-AC36 | 递归树 | 父子结构、失效引用检查和生成快照符合冻结规则。 |
