# Iteration 15 — 真实章节规划与候选批次闭环

## 1. 状态

- 当前状态：`data_model_frozen_cf_15_01b`
- UI 基线：2026-07-23 上传的 `stitch_acf_iteration_15_full_exact_shell_migration`
- P0 继承基线：`development-inputs/p0/iterations/iteration-05-chapter-planning`
- 前置完成：Iteration 13 项目工作流绑定、Iteration 14 WorkflowRun Runtime
- 外部执行依赖：`CF-14-N8N-Integration` 或等价的真实执行适配层必须在最终真实生成验收前完成；Iteration 15 不重复建设通用执行器基础设施。

## 2. 业务目标

把 P0“模拟生成章节规划”升级为真实异步章节规划闭环：

1. 用户设置完整生成、补充后续章节或指定范围重生成条件；
2. 系统执行无副作用预检；
3. 用户确认后创建 `WorkflowRun(stage=chapter_planning)`；
4. 真实执行结果经过 Schema 校验和领域事务后形成独立候选批次；
5. 用户查看、编辑、对比、采用或放弃候选；
6. 采用候选只进入“当前章节”并保持待确认，不直接确认、不直接进入正文生产；
7. 用户继续复用 P0 的编辑、删除和批量确认能力；
8. 只有已确认章节可以进入 Iteration 16 正文生产。

## 3. 核心业务原则

本节及本文件的详细状态机以 `business-rules.md`（`frozen_cf_15_01a`）为准；实体、索引、Migration 终态和事务边界以 `data-model.md`、`transaction-and-migration-design.md`（`frozen_cf_15_01b`）为准；完整 Schema 与错误码留给 01C。

### 3.1 当前章节与候选批次分离

- `ChapterPlan` 继续表示项目当前章节规划。
- 工作流输出不得直接覆盖 `ChapterPlan`。
- 每次成功输出形成一个独立 `ChapterPlanCandidateBatch`。
- 候选采用后才创建或替换当前章节。
- 被替换章节必须保留修订历史和来源追溯。

### 3.2 预检无副作用

预检只读取并校验：

- 项目策划、设定、素材、伏笔；
- 递归故事线树与失效引用；
- 项目工作流绑定、配置和连接；
- 当前活跃运行冲突；
- 输入契约、目标章节范围和版本。

预检阶段不得：

- 创建 WorkflowRun；
- 调用 n8n、LLM 或其他外部执行器；
- 创建候选批次；
- 修改当前章节。

### 3.3 运行与结果写入分离

- 用户在预检通过后确认，才创建 WorkflowRun。
- `stage` 和 `triggerSource=manual` 由服务端固定。
- WorkflowRun 负责执行生命周期与安全快照。
- Iteration 15 负责把规范化章节规划结果原子写入候选批次。
- 输出 Schema 非法或事务失败时，不允许写入部分候选。

### 3.4 采用不等于确认

- 采用候选：进入当前章节，状态为 `pending_confirmation`。
- 确认章节：复用 P0 确认 API，状态变为 `confirmed`。
- 进入正文生产：仅对 `confirmed` 章节开放。
- 批量采用逐项独立事务并返回逐项结果；冲突或 stale 候选绝不静默跳过或覆盖。

### 3.5 故事线职责

- 故事线保持递归树结构。
- 选择父节点时可在生成输入中包含其子分支。
- 故事线页面不维护可编辑章节范围。
- 章节关联数量、已确认数量、规划中数量和候选推荐数量为只读统计。
- 具体章节落点由当前章节和候选结果产生。

## 4. 用户闭环

### 4.1 发起真实章节规划

章节规划工作区
→ 点击“生成章节规划”
→ 选择生成模式、目标范围、重点故事线、上下文与附加要求
→ 执行预检
→ 查看通过、警告或阻断报告
→ 确认创建任务
→ 创建 queued WorkflowRun
→ 返回工作区或打开运行详情
→ 页面刷新后从持久化运行恢复状态。

### 4.2 处理候选批次

WorkflowRun 成功
→ 规范化结果校验
→ 原子创建候选批次及候选
→ 打开候选批次列表
→ 进入批次详情
→ 查看差异、编辑候选、单个采用或批量采用
→ 无冲突候选进入当前章节
→ 冲突候选进入重新对比、编辑或放弃流程
→ 批次状态和数量更新。

### 4.3 放弃与失败恢复

- 运行失败：保留 WorkflowRun 和安全错误，不创建候选批次。
- 输出非法：不写入任何候选，允许从运行详情重试。
- 候选过期：禁止直接采用，要求重新对比、编辑或放弃。
- 放弃批次：只终止未采用候选；已采用章节不回滚；运行和来源记录保留。
- 页面刷新：重新读取 Run、批次和当前章节，不依赖内存状态。

## 7. 与相邻迭代的边界

### 7.1 继承 Iteration 13

复用项目 `chapter_planning` 工作流绑定，不在业务页面临时切换工作流。

### 7.2 继承 Iteration 14

复用：

- WorkflowRun 创建、详情、Event、Cancel、Retry；
- 持久化幂等；
- 乐观锁；
- 绑定快照与脱敏；
- 流程中心列表和运行详情。

Iteration 15 不复制第二套 WorkflowRun。

### 7.3 与 CF-14-N8N-Integration 的边界

通用 n8n 传输、Worker、回调和外部运行状态同步属于独立执行集成任务。Iteration 15 只定义章节规划输入构建、预检、规范化结果 Schema 和候选落库。

### 7.4 与 Iteration 16 的边界

Iteration 15 结束于章节被用户确认并具备正文生产资格。正文生成、ContentVersion 创建和编辑器真实生成属于 Iteration 16。

## 8. 明确不在范围

- 不直接覆盖当前章节；
- 不自动确认候选；
- 不自动进入正文生产；
- 不在故事线页面编辑章节范围；
- 不建设 n8n 可视化编辑器；
- 不实现多执行平台路由、费用大盘或自动模型路由；
- 不实现正文生成、审核或重写；
- 不要求从当前数据库版本回滚到历史版本。
