# Iteration 15 原型评审与开发修正规则

## 1. 评审结论

新版 21 Frame 原型可作为 Iteration 15 的开发输入，覆盖度显著高于旧版 5 Frame：

- 旧版只覆盖工作区、生成抽屉和三个通用状态；
- 新版补齐预检、创建成功、候选批次、候选编辑、差异、批量采用、放弃、版本冲突和故事线职责修正；
- 业务闭环从“创建 Run”扩展到“候选安全落库并由用户采用、确认”。

旧版 Iteration 15 原型与文档不足以支撑新版业务，必须整体替换，而不是追加几个页面。

## 2. 已确认的产品语义

1. 工作流生成结果只进入候选批次。
2. 候选采用后进入当前章节，状态为待确认。
3. 采用、确认、进入正文生产是三个独立动作。
4. 预检不创建任务、不调用 n8n 或 LLM。
5. 输出非法或提交失败时零候选写入。
6. 候选基线过期时禁止直接覆盖。
7. 放弃批次不撤销已经采用的章节。
8. 故事线是递归树，章节关联统计只读，不拥有可编辑章节范围。

## 3. 必须在契约冻结阶段确认

### 3.1 真实执行依赖

Iteration 14 当前只提供 Runtime 和 Unavailable Executor。新版 UI 明确需要真实执行和连接可用性预检。

推荐边界：

- `CF-14-N8N-Integration` 继续作为独立通用执行集成；
- Iteration 15 负责章节规划预检、输入构建、结果 Schema 和候选入库；
- CF-15 最终 E2E 依赖执行集成完成；
- 不在章节规划模块重新实现 n8n Transport、Worker 或 Callback。

### 3.2 活跃运行锁

`P15_C3_PREFLIGHT_BLOCKED` 表示当前章节规划 Run 会锁定新任务。需冻结：

- 是否限制每个项目每个 stage 同时仅一个 queued/running Run；
- Cancel 后何时释放；
- Retry 是否受同一锁；
- 冲突错误码和返回的 activeRunId。

### 3.3 生成目标

需冻结三种模式的参数：

- 完整：前 N 章；
- 补充：从当前最大章节号后生成 N 章；
- 范围：startChapter–endChapter。

需明确最大数量、已有章节处理和章节号空洞规则。

### 3.4 批量采用事务

原型允许“仅采用无冲突的 11 章”，因此：

- 冲突项必须排除；
- 非冲突项是否全成功或逐项结果需冻结；
- 不得因一个冲突阻止用户处理所有无冲突项；
- 不得静默部分失败。

## 4. 原型中的实现修正

### 4.1 App Shell

- 所有页面复用当前 ACF 框架。
- `P15_C1_RUNNING_DATA_VARIANT` 中 Basic Plan 只是源数据变体，不得修改全局计划卡。
- 禁止第二侧栏、暗色顶栏或新的页面层级。

### 4.2 中文和术语

源 HTML 中存在：

- `RUNNING`
- `Batch Candidates List`
- `Ch 21`
- Material Icon 名称

实现必须使用现有图标组件和中文业务文案。技术标识可以保留英文。

### 4.3 连接状态

Iteration 14 中 `not_connected/disabled` 暂不阻止绑定和 CreateRun；新版预检要求真实执行可用性。

不得在前端自行把旧元数据当作真实连通性。连接可用性必须由执行集成后的冻结契约定义并由服务端返回。

### 4.4 C1 多版本

`c1_1` 至 `c1_6` 是同一页面的状态和数据变体：

- 不创建多个路由；
- 不复制多套表格组件；
- 通过统一组件和 fixture/真实状态覆盖；
- 不把样例统计数字写死。

### 4.5 候选与当前章节

- 批次详情中的“当前章节”是对比基线，不是候选本身。
- 编辑候选不得直接更新当前章节。
- 采用动作才写当前章节。
- 当前章节编辑和批量确认继续复用 P0 能力。

### 4.6 故事线

`P15_S1_STORYLINE_RELATION_READONLY` 只删除故事线对章节范围的错误职责：

- 不重做故事线 CRUD；
- 不把候选章节直接挂成故事线的所有权关系；
- 统计由章节和候选关系聚合。

## 5. 质量风险

| 风险 | 影响 | 控制 |
|---|---|---|
| 真实执行集成未完成 | 无法形成真实候选 | 将 CF-14-N8N-Integration 作为最终 E2E 前置门禁 |
| 候选直接复用 ChapterPlan | 生成结果覆盖用户数据 | 独立 Batch/Candidate 模型 |
| 没有章节修订历史 | 替换后无法追溯和比较 | 新增不可变 ChapterPlanRevision |
| 预检与创建状态漂移 | 检查通过后配置已变化 | preflightToken + 创建时关键项复检 |
| 批量采用语义不清 | 部分写入或误覆盖 | 冻结逐项结果和事务边界 |
| 故事线范围职责回流 | 数据模型耦合 | 仅保留递归树和只读统计 |
