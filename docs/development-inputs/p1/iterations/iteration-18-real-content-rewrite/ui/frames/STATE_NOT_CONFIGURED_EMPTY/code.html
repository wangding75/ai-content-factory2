code Developer Behavior

  • 01

    State Distinction: System must explicitly distinguish between unconfigured (never set), invalid (broken reference), disabled (turned off), loading (pending state), and permission-denied states.

  • 02

    Preflight Guarding: Never allow run initiation or generation workflows to trigger if preflight configuration checks fail. The "Run" button must be visually disabled and explain why.

  • 03

    Navigation Preservation: Empty states should occupy the content canvas only. Global navigation and sub-navigation must remain interactive to allow user escape paths.

  • 04

    Recheck Logic: Any "Refresh" or "Reconnect" action must perform a silent state check against the API without creating a new run record or session.
