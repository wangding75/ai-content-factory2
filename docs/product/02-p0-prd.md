# AI Content Factory 2.0｜P0 产品需求文档（PRD）

## 1. 版本信息

| 项 | 值 |
|---|---|
| 产品版本 | 2.0 P0 |
| 内容类型 | Novel Pack |
| 工作流能力 | 内置 Mock Provider |
| 真实 AI | 关闭 |
| 外部工作流 | 关闭 |
| 发布能力 | 关闭 |
| 目标形态 | 单用户 Creator SaaS 骨架 |

## 2. 产品目标

1. 建立小说内容从项目创建到重写版本的完整生产闭环。
2. 验证全局资产和项目用途分离的模型。
3. 验证内容版本、审核报告和工作流运行可追踪。
4. 验证 Mock Provider 可以在无真实 AI 的情况下驱动全部业务状态。
5. 建立可扩展到其他内容类型、真实 AI 和外部工作流的工程基础。

## 3. 核心功能范围

### 3.1 首页与项目

- 首页展示项目数量、素材数量、作品数量和最近项目。
- 项目列表展示小说项目，支持进入项目。
- 创建项目必须填写名称和类型；P0 类型固定为 `novel`。
- 创建成功进入同一项目 ID 的项目概览。
- 项目概览展示基础信息、当前阶段、进度摘要和快捷入口。

### 3.2 项目策划

- 编辑故事前提、目标受众、风格、目标和限制条件。
- 保存后刷新仍存在。
- 项目策划是章节规划和正文生成的输入之一。

### 3.3 素材资产

P0 支持：人物、世界观、地点、组织、道具、参考资料。

- 项目内创建素材时，同时创建全局 Material 和 ProjectMaterialUsage。
- 可从全局素材中选择并绑定到当前项目。
- 编辑素材本体影响所有引用项目。
- 编辑项目用途只影响当前项目。
- 解除绑定只删除项目用途，不删除全局素材。
- 项目与同一素材只允许存在一个有效 Usage。

### 3.4 故事线与伏笔

- 创建主故事线。
- 在任意故事线下创建子故事线。
- 展示树形关系和排序。
- 创建伏笔，记录种下故事线、回收故事线和计划章节。
- P0 支持伏笔 `planned`、`planted`、`paid_off` 状态，但不自动推断状态。

### 3.5 章节规划

- 使用 Mock Provider 生成若干章节规划候选。
- 候选可以编辑、删除和选择。
- 只有 `pending_confirmation` 候选可编辑、删除和确认。
- 确认后状态为 `confirmed`。
- 确认规划不自动创建正文；用户必须单独点击“进入正文生产”。

### 3.6 正文与审核

- 为已确认章节创建 ContentItem 和首个版本。
- 支持 Mock 生成正文和人工编辑。
- 保存草稿后刷新仍存在。
- 提交 Mock 审核后创建 ReviewReport、ReviewFinding 和 Recommendation。
- 审核只读取指定 ContentVersion，不允许覆盖正文。

### 3.7 重写与作品

- 从审核结果创建 Mock 重写任务。
- 重写必须创建新的 ContentVersion，保留父版本。
- 新版本默认不自动成为 current version。
- 新版本不自动再次审核，不自动发布。
- 项目作品页聚合当前项目内容、版本和审核状态。
- 全局作品页聚合全部项目作品。

### 3.8 全局 Lite 页面

- 全局素材：展示全局素材及引用项目。
- 全局作品：展示作品及源项目。
- 流程中心：只展示内置 Mock 流程及运行记录。
- 设置：展示能力状态；真实 AI、外部工作流和发布适配器必须标记为未配置或未开放。

## 4. 通用交互要求

每个数据页面必须实现：

- 加载态。
- 空态。
- 成功态。
- 可恢复错误态。
- 表单校验态。
- 不可用能力的禁用态。
- 关键破坏性操作的确认态。

写操作成功后必须使用服务端返回的真实 ID 和状态更新页面，不允许前端伪造成功。

## 5. 通用数据与状态规则

- 所有持久化实体具有 ID、创建时间、更新时间。
- 所有核心写操作生成 AuditLog。
- API 统一返回 `data` 和 `request_id`。
- 状态改变必须在数据库事务内完成。
- 不允许通过 UI 跳过领域状态机。
- 生成、审核和重写必须创建 WorkflowRun。
- P0 只允许 `provider_key=mock`。

## 6. 非功能要求摘要

- 本地开发环境必须通过 Docker Compose 启动 PostgreSQL 和 Redis。
- 核心 API 需要结构化日志和 request_id。
- 迁移可从空库重复执行。
- Web 和 API 的类型、字段和状态与契约一致。
- 每个迭代必须包含 Domain、Application、Repository、HTTP、Web 和 E2E 测试。
- P0 全链路验收不能只做页面打开或 HTTP 200 冒烟。

## 7. 明确不做

- 登录、团队和权限中心。
- 订阅、计费和额度。
- 真实 AI 模型调用。
- n8n、Coze、ComfyUI 实际连接。
- 发布平台连接。
- 多内容类型实现。
- 删除项目和删除全局素材。
- 多人审核、审批流和实时协作。
- 完整版本 Diff 专页。
