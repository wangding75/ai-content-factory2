# AI Content Factory 2.0｜产品总览

## 1. 产品定位

AI Content Factory 2.0 是面向个人内容创作者的内容生产操作系统。它不是单点的 AI 写作工具，而是将以下能力组织成可追踪、可复用、可扩展的生产系统：

```text
项目 + 资产 + 内容类型能力包 + 工作流 + 审核 + 版本 + 外部适配
```

P0 只以小说内容为场景，验证完整闭环和工程架构，不验证真实模型效果。

## 2. 目标用户

### 核心用户：个人小说创作者

典型特征：

- 同时维护一个或多个小说项目。
- 需要长期维护人物、地点、组织、世界观、道具和参考资料。
- 需要让故事线、章节规划、正文和审核结果保持可追踪。
- 希望使用 AI 提效，但不接受生成内容覆盖原稿或丢失版本。
- 当前常在文档、表格和多个 AI 工具之间切换，信息分散。

### 后续用户

- 内容工作室编辑。
- 短剧、图文、视频脚本等其他内容类型创作者。
- 需要连接 n8n、Coze、ComfyUI 等外部工作流的高级用户。

这些后续用户不进入 P0 实现范围，但架构必须为其保留扩展点。

## 3. 核心问题

1. **资产分散**：人物和世界观重复维护，难以跨项目复用。
2. **过程断裂**：策划、故事线、章节、正文和审核分散在不同工具。
3. **结果不可追踪**：生成和重写后缺少输入、输出、版本和来源记录。
4. **AI 侵入业务**：业务流程容易被某个模型或平台绑定。
5. **状态不清晰**：候选、确认、草稿、审核和版本关系模糊。

## 4. 价值主张

| 价值 | 产品实现 |
|---|---|
| 项目化生产 | 项目作为完整生产上下文，承载策划到作品的闭环 |
| 资产复用 | Material 全局唯一，项目通过 Usage 配置用途 |
| 过程可控 | 生成先形成候选，必须选择、编辑和确认 |
| 版本安全 | 审核不覆盖正文，重写总是创建新版本 |
| 能力可替换 | Content Pack 与 Workflow Provider 隔离类型和执行差异 |
| 结果可追踪 | WorkflowRun、AuditLog、ReviewReport 和版本链路完整保留 |

## 5. P0 成功定义

P0 成功不是“页面完成”，而是以下业务闭环可以重复执行：

```text
创建小说项目
→ 完成项目策划
→ 创建或复用素材
→ 创建故事线与伏笔
→ 生成并确认章节规划
→ 创建和编辑正文
→ 模拟审核
→ 创建重写版本
→ 在项目作品与全局作品中查询结果
```

同时满足：刷新后数据存在、状态机正确、版本不丢失、失败分支无脏数据、全部核心需求有自动化验收。
