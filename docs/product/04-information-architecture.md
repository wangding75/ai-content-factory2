# AI Content Factory 2.0｜信息架构与导航

## 1. 全局信息架构

```text
AI Content Factory
├── 首页
├── 项目
│   ├── 项目列表
│   ├── 创建项目
│   └── 项目工作区
├── 素材
│   └── 全局素材
├── 作品
│   └── 全局作品
├── 流程
│   └── 内置模拟流程
└── 设置
    └── 能力与集成状态
```

## 2. 项目工作区

```text
项目概览
├── 策划
├── 素材
├── 故事线
├── 章节
├── 审核
└── 作品
```

项目工作区必须始终保留 projectId 上下文。所有项目级页面都必须能返回项目概览，并显示当前项目名称。

## 3. 导航原则

1. 全局导航切换业务聚合视图。
2. 项目导航切换同一项目内的生产阶段。
3. 模态框和抽屉只承载短任务，不承载完整业务子系统。
4. 创建成功进入新对象的真实详情或工作区。
5. 取消操作返回来源页且不产生数据。
6. 列表刷新和浏览器返回后，筛选与项目上下文不得错误串联。

## 4. 页面链路来源

完整链路以以下冻结文件为准：

```text
docs/development-inputs/p0/baselines/p0-link-map.md
```

完整 Frame HTML 与截图位于：

```text
docs/development-inputs/p0/iterations/*/ui/frames/
```
