# AI Content Factory 2.0｜业务总览

## 1. 业务定位

通过统一项目容器、可复用资产、内容能力包和工作流执行层，降低创作者组织复杂内容生产过程的成本，并为后续真实 AI、外部流程和多内容类型建立平台基础。

## 2. P0 价值链

```text
项目建立
→ 策划沉淀
→ 资产创建与复用
→ 叙事结构设计
→ 章节候选与确认
→ 正文生产
→ 质量审核
→ 重写与版本沉淀
→ 项目/全局作品聚合
```

## 3. 业务角色

| 角色 | P0 权限与职责 |
|---|---|
| 创作者 | 创建和维护全部业务对象，确认规划，编辑正文，发起审核与重写 |
| 系统 | 执行状态校验、持久化、审计、Mock 工作流和聚合查询 |
| Mock Provider | 返回确定性或可重复的模拟结果，驱动真实业务状态 |
| 外部 Provider | P0 仅定义适配边界，不可执行 |

## 4. 业务能力域

- 项目管理。
- 项目策划。
- 全局素材与项目用途。
- 叙事结构与伏笔。
- 章节规划。
- 内容与版本。
- 审核与建议。
- 工作流运行。
- 项目作品和全局作品聚合。
- 能力与集成状态。

## 5. 业务真值原则

- Project 是生产上下文，不是所有对象的物理所有者。
- Material 是全局资产真值；ProjectMaterialUsage 是项目关系真值。
- ContentVersion 是正文快照真值；ReviewReport 只引用版本。
- WorkflowRun 是执行过程真值，但不替代领域对象。
- 聚合页面是读模型，不应复制业务真值。
