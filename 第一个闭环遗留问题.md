## Major 1：API 校验只检查 `data` 存在，无法证明固定数据被正确读取

文件：

- `scripts/test-data/09-f6e-unified-p0.ps1`

当前多数请求只执行：

```powershell
if ($null -eq $result.data) {
    throw "API did not return an envelope data value for $path"
}
```

这只能证明接口返回了带 `data` 的响应。以下情况仍会错误通过：

- `data.items` 为空；
- 返回了其他项目的数据；
- 固定素材、故事线、章节、作品或工作流未返回；
- 关联关系丢失；
- 状态值错误；
- API 映射字段缺失。

### 必须修复

为每类关键 API 增加确定性断言，至少覆盖：

1. 项目工作区：
   - 项目 ID 等于固定项目 ID；
   - planning 存在；
   - `selling_points`、`plot_summary`、`emotional_tone` 能从 API 正确读取。

2. 项目素材：
   - 固定项目至少返回预期的 4 条使用关系；
   - 共享素材存在；
   - 未绑定全局素材不应出现在项目素材中。

3. 故事线：
   - 3 条记录；
   - 1 条主线、2 条子线；
   - 子线 `parent_id` 指向固定主线。

4. 伏笔：
   - 3 条记录；
   - `planned`、`planted`、`paid_off` 三种状态均存在。

5. 章节规划：
   - 3 条记录；
   - 2 条 `confirmed`、1 条 `pending_confirmation`；
   - 章节与故事线、素材、伏笔关联可读取。

6. 作品、正文、审核：
   - 固定 content item、version、review report 存在；
   - 审核结论为 `pass`，分数为 92；
   - finding 与 recommendation 可读取。

7. 工作流：
   - 固定 generate/review 两条 run 均存在；
   - 状态均为 `succeeded`。

## Major 3：`Exercise` 未断言执行前后普通数据总量保持一致

文件：

- `scripts/test-data/09-f6e-unified-p0.ps1`

当前逻辑记录：

```powershell
$before = Invoke-Scalar 'SELECT count(*) FROM projects'
...
$after = Invoke-Scalar 'SELECT count(*) FROM projects'
Write-Host "Exercise passed. Baseline projects=$before; final projects=$after..."
```

但没有断言最终数量。即使重复加载或清理误伤了其他项目，只要 ordinary probe 那一条仍存在，`Exercise` 也可能输出 PASS。

### 必须修复

至少增加：

```powershell
if ([int]$after -ne ([int]$before + 5)) {
    throw "Project count drifted: baseline=$before final=$after expected=$([int]$before + 5)"
}
```

更稳妥的方式是：

- 在 Exercise 开始时记录非 F6E 项目的 ID 集合或数量；
- 最后断言非 F6E 项目集合/数量完全不变；
- 对 materials 等会被脚本清理的核心表做同类保护；
- 保留现有 ordinary protection probe。

## Minor 2：非法日期值没有明确兜底

文件：

- `apps/web/src/features/planning-materials/components/project-workspace-frame.tsx`

### 建议修复

如果正式 API 契约严格保证 `updated_at` 为合法时间，该项为非阻塞 Minor。
